# 🔧 封面图片路径修复说明

## 📋 问题描述

### 问题现象
在管理后台设置封面图片时，保存的封面路径是管理后台的预览路径：
```
/posts-assets/2026-03-28-/2.jpg
```

这个路径在管理后台可以正常显示，但在博客站点中无法访问，导致封面图片无法显示。

---

### 问题原因

**管理后台路径 vs 博客站点路径：**

| 环境 | 图片路径格式 | 用途 |
|------|-------------|------|
| **管理后台** | `/posts-assets/[文章名]/[文件名]` | 用于管理后台预览和管理 |
| **博客站点** | `/[文章名]/[文件名]` | 用于博客站点实际访问 |

**根本原因：**
- 管理后台使用 Express 静态文件服务，将 `source/_posts` 目录映射到 `/posts-assets` 路径
- 博客站点（Hexo）直接使用 `public` 目录下的文件结构
- 设置封面时直接使用了管理后台的预览路径，而不是博客站点的实际路径

---

## ✅ 解决方案

### 修复内容

修改 `admin/public/index.html` 中的 `setAsCover()` 函数，自动将管理后台路径转换为博客站点路径。

**修复前：**
```javascript
function setAsCover(imagePath) {
    const coverInput = document.getElementById('postCover');
    if (coverInput) {
        coverInput.value = imagePath;  // ❌ 直接使用管理后台路径
        showToast('封面设置成功');
        closeImageSelector();
    }
}
```

**修复后：**
```javascript
function setAsCover(imagePath) {
    const coverInput = document.getElementById('postCover');
    if (coverInput) {
        // 将管理后台预览路径转换为博客站点的实际路径
        // /posts-assets/[文章名]/[文件名] -> /[文章名]/[文件名]
        let blogPath = imagePath;
        if (imagePath.startsWith('/posts-assets/')) {
            // 提取文件名部分
            const parts = imagePath.split('/');
            if (parts.length >= 4) {
                const postName = parts[2];
                const fileName = parts[3];
                blogPath = `/${postName}/${fileName}`;
            }
        }
        coverInput.value = blogPath;  // ✅ 使用转换后的博客站点路径
        showToast('封面设置成功');
        closeImageSelector();
    }
}
```

---

### 转换逻辑

**路径转换示例：**

```
管理后台路径：/posts-assets/2026-03-28-/2.jpg
                      ↓ 转换
博客站点路径：/2026-03-28-/2.jpg
```

**转换步骤：**
1. 检查路径是否以 `/posts-assets/` 开头
2. 分割路径为数组：`['', 'posts-assets', '2026-03-28-', '2.jpg']`
3. 提取文章名（索引 2）：`2026-03-28-`
4. 提取文件名（索引 3）：`2.jpg`
5. 组合成博客站点路径：`/2026-03-28-/2.jpg`

## 💡 技术细节

### Hexo 的图片处理机制

**源文件结构：**
```
source/_posts/
├── hello-world.md
└── hello-world/
    └── image1.jpeg
```

**生成后结构：**
```
public/
├── hello-world/
│   └── index.html
└── hello-world/
    └── image1.jpeg
```

**访问路径：**
```
/hello-world/image1.jpeg
```

---

### 管理后台的静态文件服务

**server.js 配置：**
```javascript
// 将 /posts-assets 映射到 source/_posts 目录
app.use('/posts-assets', express.static(POSTS_DIR));
```

**为什么需要这个映射？**
- 管理后台需要访问未生成的源文件
- Hexo 的 `hexo generate` 还未执行
- 直接在 `source/_posts` 目录操作文件

---

### 为什么封面路径需要转换？

**管理后台保存的 Front Matter：**
```yaml
cover: /posts-assets/2026-03-28-/2.jpg  ❌ 错误
```

**Hexo 生成时会查找：**
```
source/posts-assets/2026-03-28-/2.jpg  ❌ 不存在
```

**正确的 Front Matter：**
```yaml
cover: /2026-03-28-/2.jpg  ✅ 正确
```

**Hexo 会找到：**
```
source/_posts/2026-03-28-/2.jpg  ✅ 存在
```
