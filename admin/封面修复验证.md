```

✅ 修复后：/2026-03-28-/2.jpg  (正常显示)
```

| 部分 | 管理后台预览路径 | 博客站点实际路径 |

| 文章名 | `2026-03-28-` | `2026-03-28-` |
| 文件名 | `2.jpg` | `2.jpg` |
| 完整路径 | `/posts-assets/2026-03-28-/2.jpg` | `/2026-03-28-/2.jpg` |

## 🔍 故障排查

### 问题 1：路径仍然包含 /posts-assets/

**可能原因：**
- 浏览器缓存未更新
- JavaScript 文件未刷新

**解决方法：**
```
强制刷新页面：
Windows: Ctrl + Shift + R 或 Ctrl + F5
Mac: Cmd + Shift + R
```

或者清除浏览器缓存后重试。

---

### 问题 2：封面在本地显示但在线上无法显示

**可能原因：**
- 图片文件未正确生成到 public 目录
- Hexo 配置问题

**检查步骤：**
```bash
# 1. 检查生成的文件
ls public/[文章名]/[文件名]

# 2. 如果不存在，重新生成
npx hexo clean
npx hexo generate

# 3. 再次检查
ls public/[文章名]/[文件名]
```

---

### 问题 3：Markdown 中的图片也不显示

**注意：** 这个修复只影响封面图片，不影响文章内容中的图片。

文章内图片应该使用：
```markdown
{% asset_img image1.jpeg %}
```

Hexo 会自动处理这些标签。

## 💡 使用技巧

### 手动修正旧文章的封面

如果有旧文章的封面路径是错误的，可以手动修正：

**方法一：在管理后台修改**
```
1. 编辑文章
2. 找到封面输入框
3. 手动删除 "/posts-assets" 部分
4. 保存文章
```

**方法二：直接编辑 Markdown 文件**
```yaml
# 修改前
cover: /posts-assets/2026-03-28-/2.jpg

# 修改后
cover: /2026-03-28-/2.jpg
```

---

### 批量检查工具（开发者工具）

在管理后台按 F12 打开开发者工具，在 Console 中运行：

```javascript
// 检查当前封面路径格式
const coverInput = document.getElementById('postCover');
if (coverInput && coverInput.value) {
    console.log('当前封面路径:', coverInput.value);
    if (coverInput.value.startsWith('/posts-assets/')) {
        console.warn('⚠️ 警告：检测到旧的封面路径格式，需要修正！');
    } else {
        console.log('✅ 封面路径格式正确');
    }
}
```
