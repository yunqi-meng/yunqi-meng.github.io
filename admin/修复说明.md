## 🚀 使用方法

### 方式一：一键启动（推荐）

双击项目根目录下的 **`start-admin.bat`**

这将自动完成：
1. 依赖安装
2. 清理旧文件
3. 生成新文件
4. 启动后台服务
5. 约 2 秒后自动打开浏览器

访问地址：**http://localhost:4007**

### 方式二：手动启动

```bash

# 2. 清理并生成（可选，但推荐）
npx hexo clean
npx hexo generate

# 3. 启动管理后台
node admin/server.js
```

## 🔧 技术细节

### 路径引用统一

所有路由文件现在都从 `../paths` 导入路径常量：

```javascript
const { POSTS_DIR, ROOT_DIR } = require('../paths');
```

而不是混用 `../server` 和 `../paths`。

### Front Matter 解析改进

**之前：** 只支持简单的内联数组 `[tag1, tag2]`

**现在：** 支持多行数组格式：

```yaml
categories:
  - 技术
  - 博客
tags:
  - Hexo
  - Node.js
```

### 部署日志类型

后端发送的日志类型包括：
- `start` - 开始部署
- `step` - 步骤切换
- `command` - 执行命令
- `output` - 命令输出
- `success` - 成功消息
- `warning` - 警告
- `error` - 错误
- `link` - 可点击链接
- `end` - 部署结束

前端根据类型渲染不同的样式和图标。

## ⚠️ 注意事项

1. **端口占用**
   - 默认使用 4007 端口
   - 如被占用，修改 `admin/server.js` 中的 `PORT` 常量

2. **Git 配置要求**
   - 本机需已安装 Git
   - 已配置 Git 用户信息
   - 已设置远程仓库 `origin`
   - 对 `main` 分支有推送权限

3. **Node.js 版本**
   - 建议使用 Node.js 18 或更高版本
   - 需要 npm 包：express, sharp 等

4. **图片上传**
   - 文章图片上传到 `source/_posts/[文章名]/` 目录
   - 独立页面图片上传到 `source/images/` 目录

## 📞 故障排查

### 问题 1：无法启动管理后台

**解决：**
```bash
# 检查端口占用
netstat -ano | findstr :4007

# 杀死占用端口的进程
taskkill /PID [进程 ID] /F

# 重新启动
node admin/server.js
```

### 问题 2：部署失败

**检查清单：**
- [ ] Git 是否已安装
- [ ] 是否能访问 GitHub
- [ ] 远程仓库名称是否为 `origin`
- [ ] 分支名称是否为 `main`
- [ ] 是否有推送权限

**查看详细错误：**
部署面板会显示详细的错误信息，根据提示进行排查。

### 问题 3：图片上传失败

**可能原因：**
- sharp 图片处理库未正确安装
- 目标文件夹权限问题

**解决：**
```bash
# 重新安装 sharp
npm uninstall sharp
npm install sharp --build-from-source
```
